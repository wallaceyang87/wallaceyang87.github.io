elasticsearch容器启动参数说明，通过-Des指定：

  • node.name 指定es节点名称，否则会随机生成节点名称。
  • cluster.name 指定es集群名称，不指定为默认的elasticsearch
  • network.host 节点绑定至容器主机名或者ip，并将此es节点发布到集群其他节点，允许ip地址，主机名，特殊值或任意组合的数组，本例中将其绑定到eth0网卡，默认是lo（local），会让集群检测失败。
  • network.publish_host 发布主机节点是唯一的接口使节点通告集群中其他节点，使这些节点可以连接到本节点，目前elasticsearch节点可以绑定多个地址，但是只允许一个地址被发布出去。如果未指定发布主机，默认选择最好的地址从network.host参数中，选取规则是由IPv4／IPv6协议栈的偏好排序，然后通过可达性排序。在本例中，我们使用的是宿主机的局域网ip，Docker容器将内部eth0网卡ip自动转换成宿主机ip，所以我们之前配置的es选项监听docker内部网络，将允许集群其他节点到达我们的节点。
  • discovery.zen.ping.unicast.hosts 本例中我们通过使用zen discovery去单播发现集群节点，此配置项是配置本节点在集群中会去发现那些集群中的其他节点
  • discovery.zen.minimum_master_nodes master 节点最小个数，本文设置为1

或者

一旦容器运行，这些环境变量就持久化在了容器内部，可以通过docker inspect查看env，也可以通过–env修改环境变量

ADD
ADD指令与COPY很相似，不过ADD命令在拷贝文件时，如果文件是压缩格式，拷贝至目标文件时会自动解压，并且ADD支持从URL下载文件拷贝至容器。

以上两种方式使用ADD指令

使用时需要注意的是src路径必须是当前build上下文目录的相对路径，不可以跳出上下文，如../somepath/somefile/,dest必须是容器内的绝对路径或者容器当前WORKDIR的相对路径(Dockerfile中之前WORKDIR指定的目录)
支持通配符的目录地址

如果<dest>以/结尾，则认为源位置指向的是一个目录，如果不以/结尾，则认为源位置是一个普通文件

COPY
与ADD相似，不能解压，不支持URL

ENTRYPOINT
ENTRYPOINT命令和CMD命令很像，只是CMD命令会在拉起容器时命令指定的指令覆盖掉，但是ENTRYPOINT不会被覆盖，docker run 命令行执行时指定的任何参数都会被当做参数传递给ENTRYPOINT指令中的命令。

VOLUME
VOLUME指令指定容器的挂载点，在docker run命令行中加入-v，将宿主机目录挂载到容器的VOLUME指令指定的目录，实现容器与宿主机间的文件处理。

Java运行时汇编代码输出

发表于 2016-11-28 | 分类于 Java |

看一些书中的示例代码，讲一些原理性的东西时，总是将Java代码转换为汇编代码来讲原理，出于好奇，也想将自己的代码转换过来看看，可惜事情总不是想想的那么简单，比如我想看一下Java中如何将volatile关键字转换后的lock头究竟是个什么样子的，折腾了半天，终于见到它的庐山真面目。
首先，我们需要HSDIS插件，jvm不提供将字节码输出成汇编代码的功能，所以我们要借助HSDIS插件来（Kenai（http://kenai.com/projects/base-hsdis）提供并得到Sun官方推荐的HotSpot VM JIT编译代码的反汇编插件）。下载该插件并拷贝到本机的/Library/Java/JavaVirtualMachines/jdk1.8.0_91.jdk/Contents/Home/jre/lib目录中。

编译测试类：

java javac InterThreadLatency.java

执行转汇编命令：

java -XX:+UnlockDiagnosticVMOptions -XX:+PrintAssembly InterThreadLatency

java -XX:+UnlockDiagnosticVMOptions '-XX:CompileCommand=print,*PongRunner.run' InterThreadLatency

0x00000001032b9299: lock addl $0x0,(%rsp) ;*putstatic ping ; - InterThreadLatency::access$002@2 (line 2) ; - InterThreadLatency$PingRunner::run@10 (line 43)

可以看到，pong属性在进行写操作时，增加了Lock指令。
